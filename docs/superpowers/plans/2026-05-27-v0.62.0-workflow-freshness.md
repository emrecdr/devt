# v0.62.0 Implementation Plan — Workflow Freshness

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.
>
> **CRITICAL — validate assumptions during implementation**: every task includes a "validate before coding" step. The existing init flow, workflow.yaml schema, and gate internals may differ from this plan's assumptions. Surface and adjust before writing code.

**Goal:** Close the architectural blind spot greenfield's v0.60.0 calibration surfaced: all mechanical assert-* gates use existence-only checks, so stale prior-workflow artifacts satisfy them. Bind every gate to workflow freshness via mtime-vs-workflow.yaml::created_at comparison (the existing `assertPreflightFresh` pattern, generalized). Plus the v0.60.0-introduced mcp-stats namespace-drift hotfix.

**Architecture:** Reset `workflow.yaml::created_at` + `workflow_id` + `workflow_type` on every `init *` (currently inconsistent). Every assert-* gate compares the artifact's mtime against `created_at - 30s grace`; older files = stale prior workflow = gate blocks. No artifact-content changes needed (no stamp prepending); just filesystem mtime + workflow.yaml::created_at.

**Tech Stack:** Node.js stdlib only, existing `path.statSync` for mtime, existing workflow.yaml regex parsing.

**Smoke target:** 666 → ~680 (+~14 gates)

**Version bump:** 0.61.0 → 0.62.0

### Validated finding (pre-dispatch validation done by controller)

`initWorkflow` currently does NOT write workflow.yaml. It reads state, scans `.devt/rules/`, loads guardrails/rubrics, and returns a JSON payload — but workflow.yaml mutations happen entirely through `state update active=true workflow_type=code_review ...` calls in each workflow's context_init bash. // Reset workflow.yaml to fresh state for this workflow. Forces workflow_id +
// created_at regeneration via the workflow_type-transition path in state.cjs.
// Without this, orchestrators that skip context_init bash inherit stale
// workflow_type / workflow_id / created_at from a prior session (greenfield
// 2026-05-27 v0.60.0 calibration: state read returned workflow_type=
// quick_implement after init review). Initialization is now mechanical at
// the CLI surface, not workflow-bash-dependent.
const workflowTypeForVerb = WORKFLOW_TYPE_BY_INIT_VERB[initVerb] || null;
if (workflowTypeForVerb) {
  const { updateState } = require("./state.cjs");
  // First, force a workflow_type transition. The updateState() logic
  // regenerates workflow_id + created_at when workflow_type changes AND
  // active=true. To guarantee the transition fires, we briefly clear
  // workflow_type then set it to the new value in a separate call —
  // OR more directly, we writeState() with a fresh shape.
  // Cleanest: call updateState({ active: true, workflow_type: workflowTypeForVerb,
  //   task: sanitizedTask, phase: "context_init", status: "in_progress",
  //   verify_iteration: 0, verdict: null }) — and the auto-stamp logic in
  //   updateState handles workflow_id + created_at.
  // To force the transition path, we ALSO need to ensure the PREVIOUS
  // workflow_type differs OR is unset. If the prior state has the same
  // workflow_type as this verb, the transition won't fire — but
  // re-initializing the same workflow type is also a legitimate use case.
  // Simplest correct approach: clear created_at + workflow_id BEFORE updateState,
  // so the "first activation" branch (line 669-672) fires.
  try {
    const fs = require("fs");
    const path = require("path");
    // nosemgrep: javascript.lang.security.audit.path-traversal.path-join-resolve-traversal
    const wfPath = path.join(projectRoot, ".devt", "state", "workflow.yaml");
    if (fs.existsSync(wfPath)) {
      let yaml = fs.readFileSync(wfPath, "utf8");
      // Strip created_at + workflow_id lines so updateState re-stamps them.
      yaml = yaml.replace(/^created_at:.*\n?/gm, "").replace(/^workflow_id:.*\n?/gm, "");
      fs.writeFileSync(wfPath, yaml);
    }
  } catch { /* fs error tolerated — updateState will create from scratch */ }
  // Now trigger updateState with the fresh workflow shape.
  updateState({
    active: true,
    workflow_type: workflowTypeForVerb,
    task: sanitizedTask,
    phase: "context_init",
    status: "in_progress",
    verify_iteration: 0,
    verdict: null,
    repair: null,
    stopped_at: null,
    stopped_phase: null,
    resume_context: null,
    memory_signal_json: null,
    scope_hint_json: null,
    scope_trust_json: null,
  });
}

### Step 2: Build a shared helper

Add a helper `isArtifactFresh(artifactPath)` that all gates call:

```javascript
// Freshness check: an artifact is fresh if its mtime is no more than
// 30 seconds OLDER than workflow.yaml::created_at. Files from prior
// workflows have mtime << current created_at → return false.
//
// Returns: { fresh: bool, reason?: string, artifact_mtime?, workflow_created_at?, age_seconds? }
//
// Auto-passes (fresh: true) when:
//   - workflow.yaml has no created_at field (legacy workflow)
//   - artifact does not exist (caller handles separately)
//
// Field rationale (greenfield 2026-05-27 v0.60.0 calibration): every
// assert-* gate used existence-only checks; stale prior-workflow
// artifacts passed silently. Binding to workflow.yaml::created_at
// (which init.cjs now resets on every init verb — see v0.62.0 T1)
// makes the gate workflow-current.
const ARTIFACT_FRESHNESS_GRACE_MS = 30 * 1000;

function isArtifactFresh(artifactPath) {
  if (!fs.existsSync(artifactPath)) {
    return { fresh: true, reason: "artifact absent (caller handles existence)" };
  }
  const dir = getStateDir();
  // nosemgrep: javascript.lang.security.audit.path-traversal.path-join-resolve-traversal
  const workflowPath = path.join(dir, "workflow.yaml");
  if (!fs.existsSync(workflowPath)) {
    return { fresh: true, reason: "no workflow.yaml — freshness check inapplicable" };
  }
  let createdAt;
  try {
    const yaml = fs.readFileSync(workflowPath, "utf8");
    const m = yaml.match(/^created_at:\s*"?([^"\n]+)"?\s*$/m);
    if (!m) return { fresh: true, reason: "workflow.yaml has no created_at stamp" };
    createdAt = new Date(m[1]).getTime();
    if (isNaN(createdAt)) return { fresh: true, reason: "workflow.yaml::created_at unparseable" };
  } catch {
    return { fresh: true, reason: "workflow.yaml read failure" };
  }
  let artifactMtime;
  try {
    artifactMtime = fs.statSync(artifactPath).mtime.getTime();
  } catch {
    return { fresh: true, reason: "artifact stat failure" };
  }
  const ageMs = createdAt - artifactMtime;
  if (ageMs > ARTIFACT_FRESHNESS_GRACE_MS) {
    return {
      fresh: false,
      reason: `artifact mtime is ${Math.round(ageMs / 1000)}s older than workflow.yaml::created_at — file is from a prior workflow`,
      artifact_mtime: new Date(artifactMtime).toISOString(),
      workflow_created_at: new Date(createdAt).toISOString(),
      age_seconds: Math.round(ageMs / 1000),
    };
  }
  return {
    fresh: true,
    artifact_mtime: new Date(artifactMtime).toISOString(),
    workflow_created_at: new Date(createdAt).toISOString(),
    age_seconds: Math.round(ageMs / 1000),
  };
}
```

Export `isArtifactFresh` so smoke gates can test it directly.

### Step 2: Add a broader WORKFLOW_EVICTABLE list + function

In state-audit.cjs, alongside GRAPHIFY_EVICTABLE:

```javascript
// Workflow-state artifacts that should be evicted on init * to prevent
// stale prior-workflow artifacts from satisfying gates. NOT a hard
// requirement — the freshness check in isArtifactFresh catches stale
// files defensively — but evicting on init removes the noise + clarifies
// state for the user.
//
// NOTE: does NOT include task outputs that follow-up workflows may need
// to consume (review.md, impl-summary.md, test-summary.md, spec.md,
// plan.md, decisions.md). Those persist across workflows by design.
const WORKFLOW_EVICTABLE = Object.freeze([
  // Gate-satisfaction markers (per-workflow)
  "scope-check-required.txt",
  "scope-check-answer.txt",
  "consolidator-ran.txt",
  "auto-curator-considered.txt",
  "reuse-analysis.md",
  "claude-mem-harvest.md",
  "claude-mem-skipped.txt",
  // Lane outputs (per-workflow)
  "lane-A.md", "lane-B.md", "lane-C.md", "lane-D.md", "lane-E.md",
  "review-lane-*.md", // pattern handled below
]);

function evictWorkflowArtifacts(opts = {}) {
  const { dryRun = false } = opts;
  const root = findProjectRoot();
  if (!root) return { ok: false, reason: "no_project_root", evicted: [], skipped: [] };
  const stateDir = path.join(root, ".devt", "state");
  if (!fs.existsSync(stateDir)) {
    return { ok: true, evicted: [], skipped: WORKFLOW_EVICTABLE.slice(), reason: "no_state_dir" };
  }
  const evicted = [];
  const skipped = [];
  // Evict literal filenames
  for (const filename of WORKFLOW_EVICTABLE) {
    if (filename.includes("*")) continue; // pattern handled separately
    const fullPath = path.join(stateDir, filename);
    if (fs.existsSync(fullPath)) {
      if (!dryRun) fs.unlinkSync(fullPath);
      evicted.push({ file: filename, dry_run: dryRun });
    } else {
      skipped.push(filename);
    }
  }
  // Evict review-lane-* glob
  try {
    for (const entry of fs.readdirSync(stateDir)) {
      if (/^review-lane-[A-Za-z0-9_.-]+\.md$/.test(entry)) {
        const fullPath = path.join(stateDir, entry);
        if (!dryRun) fs.unlinkSync(fullPath);
        evicted.push({ file: entry, dry_run: dryRun });
      }
    }
  } catch { /* readdir failure ignored */ }
  // Also call evictGraphifyArtifacts to clean those too
  const graphifyResult = evictGraphifyArtifacts({ dryRun });
  for (const item of (graphifyResult.evicted || [])) {
    evicted.push(item);
  }
  return { ok: true, evicted, skipped, dry_run: dryRun };
}
```

**Workflow Freshness — bind all assert-* gates to workflow.yaml::created_at.** Field validation of v0.60.0 (greenfield calibration) revealed that all mechanical assert-* gates used existence-only checks, so stale prior-workflow artifacts from a prior session satisfied every gate. The only gate that fired correctly (`assert-auto-curator-considered`) was the one tied to a marker the current workflow must write — every other gate had an escape clause or accepted any pre-existing file. This release binds every gate to workflow freshness via mtime-vs-workflow.yaml::created_at (generalizing the existing assertPreflightFresh pattern), ensures init * resets workflow_type/workflow_id/created_at on every invocation, adds state evict-workflow-artifacts to clean prior-workflow gate markers automatically, and hotfixes the v0.60.0-introduced mcp-stats namespace drift. Smoke: **666 → ~680 passed**, **0 failed**.

### Fixed

- **`init review` / `init workflow` / `init *` now reset workflow_type + workflow_id + created_at on every invocation**. Previously some init verbs only updated some fields, leaving stale prior-workflow values to leak (greenfield orchestrator's `state read` showed `workflow_type=quick_implement` after `init review`).
- **mcp-stats namespace drift hotfix**: workflow `mcp-stats` query lines now use unprefixed graphify tool names (`mcp__devt-graphify__*`) to match trace records. v0.60.0's C1 over-applied the prefixed-form replacement; "Graphify activity" surface returned empty for unknown duration. Trace records UNPREFIXED form regardless of how orchestrator invokes (prefixed). Convention now documented in workflow comments.

### Changed

- **All existence-checking assert-* gates now also verify artifact freshness**: artifact's mtime must be no more than 30 seconds older than workflow.yaml::created_at. Gates: assert-graphify-decision, assert-verifier-ran, assert-claude-mem-harvest, assert-scope-check-handled, assert-consolidator-dispatched, assert-auto-curator-considered, assert-reuse-analyzed. Stale prior-workflow artifacts now produce ok:false with reason "artifact mtime is Ns older than workflow.yaml::created_at — file is from a prior workflow".

### Added

- **`isArtifactFresh(path)` shared helper** in state.cjs — used by every freshness-aware gate. Returns `{fresh, reason?, artifact_mtime?, workflow_created_at?, age_seconds?}`. Exported for testing + direct programmatic use.
- **`state evict-workflow-artifacts` CLI** — cleans gate-satisfaction markers + lane files from `.devt/state/` while preserving task outputs (review.md, impl-summary.md, spec.md, plan.md, decisions.md). Called automatically by every init * verb. Manual invocation supported with `--dry-run`.
- **~14 new smoke gates (I1-I8)** covering init reset, freshness helper, gate staleness branches, evict-workflow-artifacts behavior, namespace drift convention.

### Why this matters

The greenfield calibration of v0.60.0 surfaced that every mechanical gate built across v0.58-0.61 has the same blind spot: existence is necessary but not sufficient for the artifact to represent the CURRENT workflow's work. Without freshness binding, a session can start, do nothing, and pass every gate against the prior session's leftovers. This release closes that systematically across all 7 existing gates plus the freshness helper for future ones.

### Architectural note

CON-001 substance-enforcement-gates now has TWO required properties:
1. **Existence binding**: the artifact must exist (already enforced)
2. **Freshness binding**: the artifact's mtime must postdate the current workflow.yaml::created_at within a 30s grace (new in v0.62.0)

Gates missing either property are bypassable. Both must be in place. See `docs/INTERNALS.md::Substance-Enforcement Gates` for the updated audit checklist.

### Not in this release (deferred)

- **context_init prose simplification** (greenfield orchestrator's #16) — the workflow's 250-line context_init step is hard to follow; obligations near the end get forgotten. UX gap; structurally separable from freshness binding. Tracked for a future release.
- **Knowledge-candidate aggregation to scratchpad** — lane agents append to lane outputs, orchestrator doesn't always aggregate. Worth a separate small fix.
- **Bitbucket PR-scoped tier** — still on the backlog.
